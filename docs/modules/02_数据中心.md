# 数据中心

> 本文件是 `数据中心` 模块的独立规格。Codex 开发本模块时，应同时读取 `AGENTS.md`、`docs/00_项目总览.md`、通用架构规范和对应阶段计划。

## 10. 模块二：数据中心

### 10.1 模块定位

数据中心是系统的数据入口和数据管理模块。

第一版重点做：

```text
QMT 数据源配置
账户数据同步
行情数据同步
同步任务管理
轻量数据质量检查
数据字典
```

它不负责下单，不负责策略逻辑，不负责回测计算。

### 10.2 页面结构

```text
数据中心
├── 数据来源
├── 账户数据
├── 行情数据
├── 数据同步
├── 数据质量
└── 数据字典
```

### 10.3 数据来源

#### 前端功能

展示数据源卡片。

字段：

| 字段 | 说明 |
|---|---|
| 数据源名称 | QMT |
| 连接状态 | 已连接 / 未连接 / 异常 |
| 账户 ID | 当前账户 |
| QMT 路径 | 本地路径 |
| xtquant 状态 | 已安装 / 未安装 |
| 最近连接时间 | 时间 |
| 最近检测结果 | 正常 / 异常 |

按钮：

```text
测试连接
连接数据
断开连接
环境检测
保存配置
```

#### 后端功能

服务：

```text
data_source_service.py
qmt_adapter.py
environment_check_service.py
```

职责：

1. 读取 QMT 配置。
2. 检查 xtquant 是否可导入。
3. 检查 QMT 路径是否存在。
4. 检查账户 ID 是否配置。
5. 尝试连接 QMT。
6. 返回连接状态。

#### API

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/data/sources` | 数据源列表 |
| GET | `/api/data/sources/qmt/status` | QMT 状态 |
| POST | `/api/data/sources/qmt/connect` | 连接 QMT |
| POST | `/api/data/sources/qmt/disconnect` | 断开 QMT |
| POST | `/api/data/sources/qmt/test` | 测试连接 |

### 10.4 账户数据

#### 前端功能

账户数据内部可以用二级区块，不作为主菜单。

```text
资金
持仓
委托
成交
```

资金字段：

```text
账户ID、总资产、可用资金、冻结资金、持仓市值、今日盈亏、更新时间
```

持仓字段：

```text
股票代码、股票名称、持仓数量、可卖数量、成本价、最新价、市值、浮盈亏、盈亏比例、更新时间
```

委托字段：

```text
委托时间、股票代码、股票名称、方向、价格、数量、状态、QMT订单号、来源、更新时间
```

成交字段：

```text
成交时间、股票代码、股票名称、方向、成交价、成交数量、成交金额、手续费、来源
```

交互要求：

1. 表格分页。
2. 可按代码、名称、状态、日期筛选。
3. 可刷新。
4. 可跳转交易执行。
5. 账户数据页面只读，不直接下单。

#### 后端功能

服务：`account_data_service.py`

职责：

1. 查询账户最新快照。
2. 查询持仓快照。
3. 查询委托记录。
4. 查询成交记录。
5. 不直接调用 QMT，QMT 同步由同步服务完成。

#### API

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/data/account/latest` | 最新账户资金 |
| GET | `/api/data/positions` | 持仓列表 |
| GET | `/api/data/orders` | 委托列表 |
| GET | `/api/data/trades` | 成交列表 |

### 10.5 行情数据

#### 前端功能

展示：

```text
股票基础信息
日K数据
分钟K数据
K线查看
```

股票基础字段：

```text
股票代码、股票名称、市场、证券类型、上市状态、是否ST、更新时间
```

K 线查询条件：

```text
股票代码
周期：日K / 1分钟 / 5分钟
开始日期
结束日期
```

K 线图要求：

1. 日 K 和分钟 K 均可查看。
2. 默认只加载最近一段，不加载全部历史。
3. 表格和图表都要展示。
4. 无数据时提示同步。

#### 后端功能

服务：`market_data_service.py`

职责：

1. 查询股票基础信息。
2. 查询日 K。
3. 查询分钟 K。
4. 查询最新价。
5. 对前端返回标准格式。
6. 不在查询接口里触发同步。

#### API

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/data/stocks` | 股票基础列表 |
| GET | `/api/data/kline/daily` | 查询日 K |
| GET | `/api/data/kline/minute` | 查询分钟 K |
| GET | `/api/data/quotes/latest` | 查询最新行情 |

### 10.6 数据同步

#### 前端功能

按钮：

```text
同步股票
同步账户
同步持仓
同步委托
同步成交
同步日K
同步分钟K
一键同步
停止任务
```

同步任务展示字段：

```text
任务类型、状态、进度、开始时间、结束时间、同步数量、失败数量、错误原因、操作
```

交互要求：

1. 点击同步立即返回任务 ID。
2. 页面显示进度，不阻塞。
3. 同步失败能展开技术详情。
4. 支持重试。
5. 分钟 K 同步必须选择范围，不允许默认全市场多年同步。

#### 后端功能

服务：`sync_service.py`

职责：

1. 创建同步任务。
2. 调用 QMT Adapter 拉取数据。
3. 标准化数据。
4. 批量写入 SQLite。
5. 更新同步游标。
6. 写同步日志。
7. 更新任务进度。

同步类型：

```text
stock_basic
account
position
order
trade
daily_kline
minute_kline
all
```

| POST | `/api/data/sync/trades` | 同步成交 |

| POST | `/api/data/sync/minute-kline` | 同步分钟 K |

### 10.7 数据质量

#### 前端功能

检查项：

| 检查项 | 说明 |
|---|---|
| 数据为空 | 判断关键表是否没有数据 |
| 更新时间 | 判断最新更新时间 |
| K线缺失 | 判断指定股票区间是否缺 K 线 |
| 重复数据 | 判断唯一键重复 |
| 代码格式 | 判断股票代码是否标准 |
| 委托成交 | 判断委托和成交是否匹配 |
| 同步失败 | 展示失败同步任务 |

按钮：

```text
开始检查
查看详情
复制问题
重新同步
```

职责：

1. 执行轻量质量检查。

3. 返回中文问题说明。
4. 提供修复建议。

#### API

| 方法 | 路径 | 说明 |

### 10.9 数据中心性能约束

1. 分钟 K 不能默认同步全市场多年历史。
2. 日 K 可以支持较大范围，但必须分批写入。
3. 同步过程中前端不能卡死。
4. 所有同步任务必须支持断点或游标。
5. 表格接口必须分页。
6. K 线图默认只加载有限区间。
7. 同步失败不得污染已有数据。
8. QMT 慢或失败不能影响已落库数据查询。

### 10.10 数据中心验收标准

1. 能完成 QMT 环境配置和连接测试。
2. 能同步账户、持仓、委托、成交。
3. 能同步股票基础、日 K、分钟 K。
4. 同步任务有进度、日志、失败原因。
5. 数据表格分页正常，搜索筛选正常。
6. K 线图能展示指定股票。
7. 数据质量能发现空数据、缺失数据、重复数据。
8. 数据字典能被复制给 AI 使用。
