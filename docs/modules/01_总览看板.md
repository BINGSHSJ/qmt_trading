# 总览看板

> 本文件是 `总览看板` 模块的独立规格。Codex 开发本模块时，应同时读取 `AGENTS.md`、`docs/00_项目总览.md`、通用架构规范和对应阶段计划。

## 9. 模块一：总览看板

### 9.1 模块定位

总览看板是系统首页，用于快速了解当前系统、数据、策略、交易状态。

它只做摘要展示和快捷入口，不承载复杂配置。

### 9.2 页面结构

```text
总览看板
├── 资产概览
├── 任务状态
├── 今日信号
├── 今日交易
└── 快捷操作
```

### 9.3 前端功能

#### 资产概览

展示卡片：

| 指标 | 来源 |
|---|---|
| 总资产 | account_snapshot 最新记录 |
| 可用资金 | account_snapshot 最新记录 |
| 持仓市值 | account_snapshot 或 position_snapshot 汇总 |
| 今日盈亏 | account_snapshot 或持仓浮盈亏汇总 |
| 持仓数量 | position_snapshot 当前持仓数 |
| 更新时间 | 最新账户同步时间 |

交互：

1. 点击卡片可跳转数据中心账户数据。
2. 支持手动刷新。
3. 无账户数据时提示同步账户。

#### 任务状态

展示当前运行任务：

| 任务类型 | 展示内容 |
|---|---|
| 数据同步 | 同步类型、进度、状态 |
| 策略运行 | 策略名称、进度、状态 |
| 回测任务 | 回测名称、进度、状态 |
| 备份恢复 | 任务状态 |

交互：

1. 点击任务进入详情抽屉。
2. 可停止支持取消的任务。
3. 失败任务可复制错误。

#### 今日信号

展示今日最新信号，最多 10 条。

字段：

```text
时间、策略、股票、方向、价格、金额、原因、状态、操作
```

操作：

```text
查看详情、进入信号下单、忽略
```

#### 今日交易

展示今日委托和成交摘要。

指标：

```text
已报数量、成交数量、撤单数量、失败数量、成交金额
```

#### 快捷操作

按钮：

```text
连接 QMT
同步数据
运行策略
开始回测
进入交易
环境检测
```

### 9.4 后端功能

后端服务：`dashboard_service.py`

职责：

1. 汇总账户最新数据。
2. 汇总当前任务状态。
3. 汇总今日信号。
4. 汇总今日交易。
5. 返回统一看板数据。

禁止：

1. 看板接口做大查询。
2. 看板接口触发长同步。
3. 看板接口直接调用 QMT。

### 9.5 API 设计

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/dashboard/summary` | 获取看板摘要 |
| GET | `/api/dashboard/tasks` | 获取运行中任务 |
| GET | `/api/dashboard/today-signals` | 获取今日信号摘要 |
| GET | `/api/dashboard/today-trades` | 获取今日交易摘要 |

### 9.6 数据表依赖

| 数据 | 表 |
|---|---|
| 资产 | account_snapshot |
| 持仓 | position_snapshot |
| 信号 | strategy_signal |
| 委托 | order_record |
| 成交 | trade_record |
| 任务 | runtime_task |
| 日志 | operation_log |

### 9.7 性能约束

1. 看板接口响应目标小于 500ms。
2. 看板只取摘要和最新数据，不查全量明细。
3. 今日信号最多返回 10-20 条。
4. 任务状态最多返回最近 20 个。
5. 自动刷新间隔不得小于 5 秒，避免频繁请求。

### 9.8 验收标准

1. 未连接 QMT 时能清楚提示下一步。
2. 账户同步后看板资产能更新。
3. 策略产生信号后今日信号能显示。
4. 下单后今日交易能显示委托状态。
5. 任务失败后能在看板复制错误给 AI。
