# 回测研究

> 本文件是 `回测研究` 模块的独立规格。Codex 开发本模块时，应同时读取 `AGENTS.md`、`docs/00_项目总览.md`、通用架构规范和对应阶段计划。

## 12. 模块四：回测研究

### 12.1 模块定位

回测研究用于验证 Python 策略效果。

第一版只做单策略回测，不做多策略组合、不做参数寻优、不做复杂归因。

### 12.2 页面结构

```text
回测研究
├── 新建回测
├── 回测任务
├── 绩效结果
├── 交易明细
└── 回测日志
```

### 12.3 新建回测

#### 前端功能

表单字段：

| 字段 | 说明 |
|---|---|
| 策略文件 | 选择 Python 策略 |
| 回测名称 | 默认策略名 + 日期 |
| 开始日期 | 回测开始日期 |
| 结束日期 | 回测结束日期 |
| 初始资金 | 默认 1000000 |
| 单笔金额 | 默认来自策略或系统设置 |
| 数据频率 | 日 K / 分钟 K |
| 成交方式 | 下一分钟 / 下一日开盘 / 收盘价 |
| 手续费 | 默认配置 |
| 印花税 | 默认配置 |
| 滑点 | 默认配置 |

按钮：

```text
开始回测
保存配置
检查数据
```

#### 后端功能

服务：`backtest_engine.py`

职责：

1. 创建回测任务。
2. 检查数据完整性。
3. 加载策略。
4. 加载历史数据。
5. 执行回测。
6. 计算绩效。
7. 保存结果。
8. 保存交易明细。
9. 保存收益曲线。

#### API

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/backtests` | 创建回测任务 |
| POST | `/api/backtests/{task_id}/cancel` | 取消回测 |
| POST | `/api/backtests/check-data` | 检查回测数据 |

### 12.4 A 股回测规则

必须支持：

| 规则 | 要求 |
|---|---|
| T+1 | 当日买入不可当日卖出 |
| 100 股一手 | 买入数量按 100 股取整 |
| 手续费 | 买卖计入 |
| 印花税 | 卖出计入 |
| 滑点 | 可配置 |
| 涨跌停 | 涨停买不进、跌停卖不出，按数据判断 |
| 停牌 | 停牌不可交易 |
| 信号延迟 | 信号产生后按配置的下一时间点成交 |
| 成交量限制 | 可选，避免超过成交量比例 |
| 复权处理 | 字段预留，第一版可简化 |

### 12.9 回测性能约束

1. 回测必须异步任务化。
2. 回测不能阻塞 API 主线程。
3. 第一版不做全市场多年分钟级暴力回测。
4. 分钟级回测必须限制股票池和时间区间。
5. 结果分批写入数据库。
6. 曲线数据过大时要抽样返回。
7. 回测任务必须可取消。
8. 回测前必须检查数据是否存在。

### 12.10 回测验收标准

1. 能选择 Python 策略创建回测。
2. 能检查数据是否充足。
3. 能异步运行并显示进度。
4. 能展示收益、回撤、胜率等指标。
5. 能展示交易明细。
6. 能查看回测日志。
7. 回测失败能复制错误给 AI。
8. 回测逻辑与策略信号接口一致。

## 27. 回测引擎深度规范

### 27.1 回测引擎组成

```text
DataLoader 数据加载器
StrategyRunner 策略执行器
BacktestBroker 本地撮合经纪商
MatchingEngine 撮合引擎
Portfolio 组合持仓
MetricsService 指标计算
ReportService 报告输出
```

### 27.2 数据加载器

职责：

1. 按策略和回测区间加载数据。
2. 不一次性加载无关股票全量数据。
3. 检查数据缺失。
4. 标准化字段。
5. 提供给 StrategyContext。

### 27.3 本地撮合经纪商

职责：

1. 管理现金。
2. 管理持仓。
3. 管理可卖数量。
4. 应用 T+1。
5. 应用手续费、印花税、滑点。
6. 生成本地撮合订单和成交。

### 27.4 撮合规则

第一版简化但必须清晰：

| 场景 | 规则 |
|---|---|
| 买入 | 按下一分钟/下一日配置价格成交 |
| 卖出 | 按下一分钟/下一日配置价格成交 |
| 涨停 | 买入失败或不成交 |
| 跌停 | 卖出失败或不成交 |
| 停牌 | 不成交 |
| 资金不足 | 买入数量按资金裁剪或失败，需记录原因 |
| 不足一手 | 不下单 |
| 当日买入卖出 | T+1 限制不允许 |

### 27.5 指标计算

必须计算：

```text
总收益率
年化收益率
最大回撤
胜率
交易次数
盈亏比
平均持仓天数
最终资金
```

指标必须有函数单元测试。

### 27.6 回测一致性

回测使用的策略接口必须与实盘运行一致。

禁止：

1. 回测使用一套策略逻辑，实盘使用另一套。
2. 回测忽略涨跌停和停牌。
3. 回测忽略费用。
4. 回测直接使用未来数据。
