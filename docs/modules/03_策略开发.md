# 策略开发

> 本文件是 `策略开发` 模块的独立规格。Codex 开发本模块时，应同时读取 `AGENTS.md`、`docs/00_项目总览.md`、通用架构规范和对应阶段计划。

## 11. 模块三：策略开发

### 11.1 模块定位

策略开发是 Python 策略管理和运行模块。

它只负责：

```text
策略文件管理
代码编辑
接口检查
运行调试
信号保存
版本记录
```

它不负责：

```text
下单
回测成交撮合
QMT 直接调用
无代码生成策略
```

### 11.2 页面结构

```text
策略开发
├── 策略文件
├── 代码编辑
├── 运行调试
├── 策略信号
└── 版本记录
```

2. 从示例复制。

4. 读取策略元信息。

6. 确保不会覆盖用户文件。

### 11.5 Python 策略接口规范

所有策略必须符合以下接口：

```python
class Strategy:
    name = "策略名称"
    version = "1.0.0"
    description = "策略说明"

    params = {
        "single_order_amount": 100000
    }

    def __init__(self, context):
        self.context = context

    def run(self):
        signals = []
        return signals
```

### 11.6 策略信号格式

策略返回信号必须是列表，每个元素为字典：

```python
{
    "symbol": "600000.SH",
    "name": "浦发银行",
    "action": "BUY",
    "price": 10.25,
    "amount": 100000,
    "reason": "1分钟成交额放大3.6倍",
    "signal_time": "2026-05-08 10:15:00"
}
```

字段要求：

| 字段 | 是否必填 | 说明 |
|---|---|---|
| symbol | 是 | 标准股票代码 |
| name | 否 | 股票名称，可由系统补齐 |
| action | 是 | BUY / SELL / WATCH |
| price | 是 | 参考价格 |
| amount | 否 | 建议金额 |
| reason | 是 | 触发原因 |
| signal_time | 否 | 不填则使用运行时间 |

### 11.7 StrategyContext 能力

策略只能通过 `context` 读取数据。

允许方法：

```python
context.get_stock_list(...)
context.get_daily_bars(symbol, start_date, end_date)
context.get_minute_bars(symbol, start_time, end_time)
context.get_latest_price(symbol)
context.get_account_snapshot()
context.get_positions()
context.get_trading_calendar(start_date, end_date)
context.log(message)
```

禁止策略直接获得：

```text
QMT 原始对象
数据库连接对象
交易下单对象
系统配置明文
文件系统任意写权限
```

#### 后端功能

服务：`strategy_runner.py`

职责：

1. 创建运行任务。
2. 在独立子进程或受控任务中执行策略。
3. 注入 StrategyContext。
4. 捕获 stdout/stderr。
5. 捕获异常。
6. 校验输出信号。
7. 保存信号。
8. 更新运行日志。

运行约束：

1. 策略运行必须有超时时间。
2. 策略运行必须支持停止。
3. 策略不能阻塞主 API 线程。
4. 策略不能直接下单。
5. 策略不能修改数据库结构。

### 11.11 策略开发性能约束

1. Monaco Editor 懒加载。
2. 策略运行必须异步任务化。
3. 策略不能一次性读取全库数据。
4. StrategyContext 必须按需查询。
5. 策略日志不能无限写入，单次运行日志要限制大小。
6. 信号批量写入。
7. 运行失败不影响其他策略和主服务。

### 11.12 策略开发验收标准

1. 能新建 Python 策略文件。
2. 能导入 Python 策略文件。
3. 能在线编辑、保存、恢复版本。
4. 能检查策略接口。
5. 能运行策略并保存信号。
6. 策略错误能显示中文说明和技术详情。
7. 策略不能直接调用交易接口。
8. 生成信号后可进入交易执行和回测研究。

## 26. 策略运行安全与隔离规范

### 26.1 基本原则

用户策略是本地 Python 代码。第一版不承诺完全沙箱化，但必须做最小隔离，避免策略误操作交易和系统资源。

### 26.2 必须隔离

1. 策略拿不到 QMT 原始连接。
2. 策略拿不到交易下单服务。
3. 策略拿不到数据库连接对象。
4. 策略通过 `StrategyContext` 读数据。
5. 策略只能返回信号。
6. 策略运行有超时。
7. 策略运行日志被捕获。
8. 策略运行失败不影响主服务。

### 26.3 建议限制

1. 策略在子进程中运行。
2. 策略工作目录限制在项目目录内。
3. 不把系统配置和账户敏感信息传给策略。
4. 对策略输出做严格校验。
5. 策略信号金额超过系统最大单笔金额时，交易执行模块再次确认或拦截。

### 26.4 明确风险提示

由于 Python 本身可执行任意本地代码，用户从 AI 或外部复制策略代码前，应在代码编辑器中检查。系统可以提供“策略代码安全提示”，但第一版不做完整恶意代码检测。
