# 06_性能优化规范

> 本文件约束性能红线、容量规划、SQLite 性能、前端懒加载、任务异步化、日志清理和交易一致性。

## 18. 性能优化与容量规划

### 18.1 性能红线

禁止以下实现：

1. 前端全量加载大表。
2. 回测接口同步等待直到结束。
3. 策略运行阻塞 API 主线程。
4. 分钟 K 默认全市场多年同步。
5. SQLite 逐条 commit。
6. 查询无索引。
7. 日志无限增长。
8. 图表一次渲染几十万点。
9. QMT 调用散落在多个模块。
10. 同一信号重复下单。

### 18.2 容量规划

| 数据 | 第一版策略 |
|---|---|
| 股票基础 | 全量同步 |
| 日 K | 可较长周期同步 |
| 分钟 K | 按股票池和时间区间同步 |
| 账户持仓 | 高频同步但数据小 |
| 委托成交 | 按日期增量同步 |
| 策略信号 | 分页展示，定期清理 |
| 回测明细 | 按任务保存，可删除 |
| 日志 | 保留天数可配置 |

### 18.3 前端性能

1. 路由懒加载。
2. 大组件懒加载。
3. 表格分页。
4. 搜索防抖 300ms。
5. 自动刷新间隔最低 5 秒。
6. 离开页面取消请求。
7. 图表数据降采样。
8. Monaco 只在代码编辑 Tab 加载。

### 18.4 后端性能

1. 读写分层。
2. 数据批量写入。
3. 查询加索引。
4. 长任务后台执行。
5. API 不返回过大 payload。
6. 导出单独任务化。
7. 任务日志分批写入。
8. QMT 慢调用有超时控制。

### 18.5 SQLite 性能

1. 启用 WAL。
2. 批量 upsert。
3. 使用唯一索引防重复。
4. 大表按 symbol/date 查询。
5. 定期清理日志和旧任务。
6. 提供数据库大小监控。
7. 必要时提供 VACUUM/维护按钮，但默认不自动频繁执行。

### 18.6 回测性能

1. 回测数据按需加载。
2. 单策略单任务。
3. 分批保存交易和曲线。
4. 支持取消。
5. 支持进度。
6. 分钟级回测必须限制范围。

### 18.7 交易性能

1. 交易接口优先级最高。
2. 下单请求不能被历史行情同步阻塞。
3. 防重复提交。
4. 委托状态同步必须及时。
5. 订单异常必须可追踪。

---

## 性能开发底线

1. 禁止前端加载全量列表。
2. 禁止后端列表接口无分页。
3. 禁止长任务阻塞 HTTP 请求。
4. 禁止分钟 K 默认全市场多年全量同步。
5. 禁止 SQLite 逐条 commit。
6. 禁止没有索引的高频查询。
7. 禁止策略运行阻塞主服务。
8. 禁止回测一次性写入巨大结果对象。
9. 禁止日志无限增长。
10. 禁止下单按钮重复提交。
