# 06_交易执行实施计划

## 本次目标

实现交易执行中心，支持手动下单、信号下单、撤单、持仓、委托、成交、执行日志。第一版保留最小交易保护，不做复杂风控中心。

- `docs/00_项目总览.md`
- `docs/03_后端架构规范.md`
- `docs/04_数据库设计规范.md`
- `docs/05_API接口规范.md`
- `docs/modules/05_交易执行.md`

```text
绕过人工确认或幂等保护提交真实委托
```

## 具体任务

1. 实现交易面板：股票代码、买卖方向、价格、数量、金额、可用资金、可卖数量。
2. 实现信号下单：从策略信号生成订单，绑定 signal_id。
3. 实现当前持仓展示。
4. 实现委托记录和撤单。
5. 实现成交记录。
6. 实现执行日志。
7. 实现订单生命周期和状态映射。
8. 实现幂等和防重复下单：按钮锁定、local_order_id、信号状态、重复请求保护。
9. 实现 QMT 交易 Adapter，并提供测试隔离交易替身用于自动化测试和离线排障。
10. 下单前确认弹窗默认开启。

## 验收标准

1. 测试隔离模式下可完成手动下单、信号下单、撤单、状态同步；真实 QMT 小额验收前不得提交真实委托。
2. 每笔订单有 local_order_id 和 qmt_order_id。
3. 信号下单后信号状态更新，重复点击不会重复下单。
4. 委托和成交状态可同步刷新。
5. 下单、撤单、失败都写执行日志和操作记录。
6. 交易接口失败时显示中文原因。
7. 不做复杂风控中心，不做审批流。


## 通用执行规则

1. 本阶段只做本文件定义的范围。
2. 先阅读 `AGENTS.md`。
3. 再阅读 `docs/00_项目总览.md`。
4. 根据需要阅读相关通用规范和模块文档。
5. 不允许新增未定义主模块。
6. 不允许引入 Redis、Celery、Docker、微服务。
7. 不允许覆盖 `strategies/user/`。
8. 完成后必须输出修改文件清单、接口清单、运行方式、测试结果。
