# UI 信息架构与视觉方向重审实施计划（2026-05-22）

## 1. 阶段定位

本计划是 UI/UX 单独立项计划，不属于普通 QA 修复。目标是解决“整体不像成熟金融桌面产品”的问题，而不是继续零散修按钮、输入框和表格边距。

本阶段只允许前端信息架构、视觉系统、页面骨架和视觉测试相关工作，不改变后端业务逻辑。

## 2. 总体目标

把当前后台从“暗色后台页面集合”推进为“统一金融交易工作台”：

- 全局骨架统一。
- 六大页面信息层级统一。
- 表格、详情、弹窗、抽屉统一。
- 主链路在视觉上可追踪。
- 保持本地轻量架构和真实 QMT 只读安全边界。

## 3. 执行边界

允许修改：

- `docs/01_UIUX设计规范.md`
- `frontend/src/theme/`
- `frontend/src/styles/`
- `frontend/src/layouts/`
- `frontend/src/components/AppShell/`
- `frontend/src/components/PageHeader/`
- `frontend/src/components/DataTable/`
- `frontend/src/components/Workspace/`
- 新增轻量前端通用组件
- 六大页面的布局层和展示层
- Playwright 视觉测试
- UI 重审报告

禁止修改：

- 后端业务逻辑。
- QMT Adapter。
- 交易执行语义。
- 回测核心算法。
- Python 策略接口。
- 六大菜单名称。
- 路由路径。
- `strategies/user/`。
- 默认自动实盘交易。

## 4. 分批执行计划

### IA-1：规范口径更新

目标：

- 正式把 UI 方向从旧的“明亮中后台”更新为“暗色金融桌面工作台”。
- 解决文档和当前用户需求冲突。

交付：

- 更新 `docs/01_UIUX设计规范.md`。
- 新增暗色金融桌面视觉章节。
- 明确 IBKR Desktop 只是参考方向。

验收：

- 文档不再与当前暗色工作台方向冲突。
- 禁止范围完整保留。

### IA-2：全局 Shell 重新定义

目标：

- 建立统一的 AppShell、左侧导航、顶部状态、底部状态、页面命令区。

交付：

- `AppShell` 结构收敛。
- `TopStatusBar` 与 `BottomStatusBar` 视觉统一。
- `PageCommandBar` 替代页面散落操作区。

验收：

- 六大菜单名称不变。
- 六大页面均能进入。
- 1280/1440/1920 视口无页面级横向溢出。

### IA-3：工作台组件体系

目标：

- 建立统一页面骨架，减少各页面各自写布局。

交付建议：

- `WorkbenchPage`
- `WorkbenchPane`
- `InspectorPanel`
- `MetricStrip`
- `StatusMatrix`

验收：

- 新组件不直接调用业务 API。
- 页面仍通过 services 层请求。
- 不新增业务模块。

### IA-4：六大页面逐页迁移

顺序建议：

1. 总览看板。
2. 数据中心。
3. 策略开发。
4. 回测研究。
5. 交易执行。
6. 系统管理。

每页迁移原则：

- 保留原有功能。
- 保留原有接口。
- 只调整展示、布局、信息层级。
- 每页迁移后跑对应 Playwright 冒烟和视觉测试。

### IA-5：表格、右侧详情、弹窗统一

目标：

- 表格列密度、固定列、操作列、分页区统一。
- 详情抽屉和右侧检查器统一。
- 交易确认、备份恢复、错误详情、回测导出等弹窗统一。

验收：

- 弹窗宽度、内容块留白、按钮区一致。
- 表格无白底泄漏、无底部异常横条。
- 操作列不撑破页面。

### IA-6：视觉回归护栏

目标：

- 用自动化防止视觉回退。

交付：

- 六大页面截图基线。
- 关键弹窗和抽屉截图基线。
- 横向溢出、浅色泄漏、控件高度、表格列宽自动检查。

验收：

- `npm run typecheck` 通过。
- `npm run lint` 通过。
- Playwright 视觉回归通过。

## 5. 验收清单

每一批完成后必须输出：

1. 本批做了什么。
2. 修改文件清单。
3. 是否新增接口。
4. 是否新增数据库表。
5. 是否触碰禁止范围。
6. 测试命令和结果。
7. 截图或自动化指标路径。
8. 下一批建议。

## 6. 不建议现在做

- 不建议为了视觉重构改后端接口。
- 不建议为了像 IBKR 而改变交易流程。
- 不建议一次性重写六大页面。
- 不建议在没有规范更新前继续大量堆 CSS。
- 不建议现在引入完整商业化设计系统。
