# 04_策略开发实施计划

## 本次目标

实现 Python 策略开发中心。策略只支持 Python 新建、导入、编辑、检查、运行、信号展示和版本记录。

## 需阅读文档

- `AGENTS.md`
- `docs/00_项目总览.md`
- `docs/01_UIUX设计规范.md`
- `docs/02_前端架构规范.md`
- `docs/03_后端架构规范.md`
- `docs/modules/03_策略开发.md`

## 允许修改范围

```text
backend/api/strategy_dev_api.py
backend/services/strategy_dev/
backend/repositories/strategy_dev/
backend/schemas/strategy_dev.py
backend/models/strategy_models.py
frontend/src/pages/StrategyDev/
frontend/src/services/strategyDev.ts
frontend/src/components/CodeEditor/
frontend/src/components/SignalTable/
strategies/examples/
```

## 禁止修改范围

```text
strategies/user/ 已存在用户策略文件不得覆盖
交易执行真实下单逻辑
条件筛选器、拖拽策略、无代码策略生成器
```

## 具体任务

1. 实现策略文件列表。
2. 实现新建 Python 策略文件。
3. 实现导入 Python 策略文件。
4. 实现 Monaco 代码编辑器和懒加载。
5. 实现保存策略与版本快照。
6. 实现策略接口检查：是否存在 `Strategy`、`run()`、返回 signals。
7. 实现策略运行任务：子进程或受控任务、超时、停止、日志捕获。
8. 实现 `StrategyContext`，提供按需读取数据方法。
9. 实现策略信号保存与展示。
10. 实现错误技术详情复制给 AI。

## 验收标准

1. 可以在页面新建、导入、编辑、保存 Python 策略。
2. 保存时自动生成版本记录。
3. 策略接口错误时给中文提示和技术详情。
4. 策略运行不阻塞页面。
5. 策略只生成信号，不能直接调用 QMT 下单。
6. 信号列表可展示、筛选、刷新、复制原因。
7. 不做筛选器、拖拽器、无代码策略生成。


## 通用执行规则

1. 本阶段只做本文件定义的范围。
2. 先阅读 `AGENTS.md`。
3. 再阅读 `docs/00_项目总览.md`。
4. 根据需要阅读相关通用规范和模块文档。
5. 不允许新增未定义主模块。
6. 不允许引入 Redis、Celery、Docker、微服务。
7. 不允许覆盖 `strategies/user/`。
8. 完成后必须输出修改文件清单、接口清单、运行方式、测试结果。
