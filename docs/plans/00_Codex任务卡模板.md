# 00_Codex任务卡模板

以后每次给 Codex 的任务，建议复制这个模板。

```text
请先阅读：
- AGENTS.md
- docs/00_项目总览.md
- docs/<相关通用规范>.md
- docs/modules/<相关模块>.md
- docs/plans/<当前阶段实施计划>.md

本次目标：
<只写本次要完成的具体目标，不要写完整项目>

允许修改范围：
<列出允许改的目录和文件>

禁止修改范围：
<列出不能动的目录、模块、技术方案>

具体任务：
1. ...
2. ...
3. ...

验收标准：
1. ...
2. ...
3. ...

测试要求：
- 后端 pytest
- 前端 npm run lint
- 前端 npm run typecheck
- 如涉及页面流程，补充 Playwright 测试

输出要求：
1. 修改文件清单
2. 新增接口清单
3. 新增或修改表结构
4. 启动方式
5. 测试结果
6. 哪些是真实实现，哪些仅为测试隔离替身
7. 是否触碰禁止范围
8. 下一步建议
```
