# 02_系统管理实施计划

## 本次目标

优先实现系统管理，因为它负责 QMT 路径、账户 ID、环境检测、日志、备份和运行监控，是后续数据中心、交易执行的基础。

## 需阅读文档

- `AGENTS.md`
- `docs/00_项目总览.md`
- `docs/03_后端架构规范.md`
- `docs/04_数据库设计规范.md`
- `docs/05_API接口规范.md`
- `docs/modules/06_系统管理.md`

## 允许修改范围

```text
backend/api/system_api.py
backend/services/system/
backend/repositories/system/
backend/schemas/system.py
backend/models/system_models.py
frontend/src/pages/SystemManagement/
frontend/src/services/system.ts

```

## 禁止修改范围

```text
strategies/user/
交易执行真实下单逻辑
回测引擎核心算法
```

## 具体任务

1. 实现基础设置：QMT 路径、账户 ID、数据库路径、启动自动连接、启动自动同步。
2. 实现环境检测任务：QMT 路径、MiniQMT 状态、xtquant 可导入、Python 兼容性、资产查询、持仓查询、行情查询、下单能力检测。
3. 实现日志中心：同步日志、策略日志、回测日志、交易日志、系统日志统一展示。
4. 实现运行监控：任务状态、数据库大小、最近错误、慢任务。
5. 实现备份恢复：立即备份、恢复备份、打开备份目录、备份记录。
6. 实现操作记录：配置变更、环境检测、备份恢复等。
7. 所有检测都必须支持测试隔离模式，便于自动化测试和离线排障；业务默认仍按真实 QMT 只读优先执行。

## 验收标准

1. 可在页面保存和读取基础设置。
2. 点击环境检测返回检测项列表和中文结果。
3. 环境检测作为任务执行并返回 `task_id`。
4. 日志页面支持分页、筛选、复制技术详情。
5. 备份页面能生成备份记录。
6. 所有失败提示中文化。
7. 不依赖真实 QMT 也能通过测试隔离替身完成自动化测试。


## 通用执行规则

1. 本阶段只做本文件定义的范围。
2. 先阅读 `AGENTS.md`。
3. 再阅读 `docs/00_项目总览.md`。
4. 根据需要阅读相关通用规范和模块文档。
5. 不允许新增未定义主模块。
6. 不允许引入 Redis、Celery、Docker、微服务。
7. 不允许覆盖 `strategies/user/`。
8. 完成后必须输出修改文件清单、接口清单、运行方式、测试结果。
