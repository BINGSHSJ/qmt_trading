# 05_回测研究实施计划

## 本次目标

实现单策略回测研究中心，支持选择 Python 策略、设置区间和资金、运行回测、查看指标、曲线、交易明细和日志。

## 需阅读文档

- `AGENTS.md`
- `docs/00_项目总览.md`
- `docs/03_后端架构规范.md`
- `docs/04_数据库设计规范.md`
- `docs/06_性能优化规范.md`
- `docs/modules/04_回测研究.md`

## 允许修改范围

```text
backend/api/backtest_api.py
backend/services/backtest_center/
backend/repositories/backtest_center/
backend/schemas/backtest.py
backend/models/backtest_models.py
frontend/src/pages/BacktestResearch/
frontend/src/services/backtest.ts
frontend/src/components/BacktestChart/
```

## 禁止修改范围

```text
strategies/user/ 不覆盖用户策略
交易执行真实下单逻辑
多策略组合
参数网格寻优
全市场多年分钟级暴力回测
```

## 具体任务

1. 实现新建回测表单：策略、区间、初始资金、单笔金额、手续费、印花税、滑点、成交方式、数据频率。
2. 实现回测任务：状态、进度、日志、取消。
3. 实现 DataLoader、StrategyRunner、BacktestBroker、MatchingEngine、Portfolio、MetricsService、ReportService。
4. 支持 A 股基础规则：T+1、100 股一手、手续费、印花税、滑点、涨跌停、停牌、信号延迟。
5. 保存回测结果、交易明细、收益曲线、回测日志。
6. 前端展示总收益率、年化收益率、最大回撤、胜率、交易次数、盈亏比、最终资金。
7. 展示收益曲线、回撤曲线、每日盈亏。

## 验收标准

1. 可以选择 Python 策略启动回测。
2. 回测作为任务执行，页面可查看进度和日志。
3. 回测结果指标正确展示。
4. 交易明细分页展示。
5. 曲线图能展示收益和回撤。
6. 回测不直接调用 QMT 实盘交易接口。
7. 回测失败有中文原因和技术详情。


## 通用执行规则

1. 本阶段只做本文件定义的范围。
2. 先阅读 `AGENTS.md`。
3. 再阅读 `docs/00_项目总览.md`。
4. 根据需要阅读相关通用规范和模块文档。
5. 不允许新增未定义主模块。
6. 不允许引入 Redis、Celery、Docker、微服务。
7. 不允许覆盖 `strategies/user/`。
8. 完成后必须输出修改文件清单、接口清单、运行方式、测试结果。
