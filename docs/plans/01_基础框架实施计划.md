# 01_基础框架实施计划

## 本次目标

创建项目基础骨架和可视化壳子，不接真实 QMT，不实现真实回测和交易。

## 需阅读文档

- `AGENTS.md`
- `docs/00_项目总览.md`
- `docs/01_UIUX设计规范.md`
- `docs/02_前端架构规范.md`
- `docs/03_后端架构规范.md`
- `docs/05_API接口规范.md`
- `docs/06_性能优化规范.md`

## 允许修改范围

```text
backend/
frontend/
docs/ 如需补充说明
start.bat
README.md
```

## 禁止修改范围

```text
strategies/user/
真实 QMT 交易接口
真实下单逻辑
复杂回测算法
```

## 具体任务

1. 创建后端 FastAPI 项目骨架。
2. 创建前端 React + TypeScript + Vite 项目骨架。
3. 实现六大四字主菜单：总览看板、数据中心、策略开发、回测研究、交易执行、系统管理。
4. 实现统一布局：左侧菜单、顶部状态栏、内容区。
5. 每个模块先用测试隔离数据展示基础页面；测试隔离数据只用于自动化测试、离线回归和开发排障，不作为业务默认数据。
6. 实现统一请求封装、错误弹窗、加载态、空状态。
7. 后端提供健康检查接口。
8. 创建本地启动脚本 `start.bat`。
9. 创建基础日志目录、数据目录、备份目录。

## 验收标准

1. 双击或运行启动脚本后，前后端能启动。
2. 浏览器可以看到六大主菜单。
3. 每个主菜单都有基础页面和测试隔离表格或卡片，页面需明确不依赖真实 QMT 才能完成基础验收。
4. 后端健康检查接口可访问。
5. 前端出现错误时有中文错误弹窗。
6. 项目目录符合总规范。
7. pytest、lint、typecheck 通过，或说明尚未初始化对应命令的原因。


## 通用执行规则

1. 本阶段只做本文件定义的范围。
2. 先阅读 `AGENTS.md`。
3. 再阅读 `docs/00_项目总览.md`。
4. 根据需要阅读相关通用规范和模块文档。
5. 不允许新增未定义主模块。
6. 不允许引入 Redis、Celery、Docker、微服务。
7. 不允许覆盖 `strategies/user/`。
8. 完成后必须输出修改文件清单、接口清单、运行方式、测试结果。

## 通用测试命令

实际命令以项目初始化后的 package/scripts 为准，第一版建议固定为：

```bash
# 后端
cd backend
pytest

# 前端
cd frontend
npm run lint
npm run typecheck
```
