# 07_整体联调实施计划

## 本次目标

完成全链路联调、性能验收、UI/UX 精修、测试隔离与真实 QMT 只读模式边界检查，确保第一版可稳定使用。

## 需阅读文档

- `AGENTS.md`
- `docs/00_项目总览.md`
- 所有通用规范
- 所有模块文档
- 已完成阶段的开发记录

## 允许修改范围

```text
全项目范围内的问题修复
UI/UX 精修
测试补充
性能优化
文档补充
```

## 禁止修改范围

```text
新增未定义主模块
引入 Redis / Celery / Docker / 微服务作为默认方案
大幅重构已稳定模块但无明确收益
覆盖 strategies/user/ 用户策略
```

## 具体任务

1. 联调主链路：启动系统 → 环境检测 → 数据同步 → 新建策略 → 运行策略 → 回测 → 信号下单 → 同步委托成交 → 总览展示。
2. 检查所有接口统一响应格式。
3. 检查所有列表分页、筛选、排序。
4. 检查所有长任务返回 task_id。
5. 检查所有错误中文化。
6. 检查所有页面加载态、空状态、错误态。
7. 检查数据库索引、WAL、同步游标、日志清理。
8. 进行本地容量压测。
9. 进行交易防重复提交测试。
10. 补充 Playwright 端到端测试。
11. 输出最终使用说明。

## 验收标准

1. 第一版主链路完整跑通。
2. 测试隔离模式无真实 QMT 也能完成自动化回归和离线排障。
3. 真实 QMT 模式可以通过环境检测后启用只读同步；真实下单仍需单独小额人工验收。
4. 页面无明显英文报错。
5. 性能红线未违反。
6. 测试通过。
7. 用户可以不看代码完成核心操作。


## 通用执行规则

1. 本阶段只做本文件定义的范围。
2. 先阅读 `AGENTS.md`。
3. 再阅读 `docs/00_项目总览.md`。
4. 根据需要阅读相关通用规范和模块文档。
5. 不允许新增未定义主模块。
6. 不允许引入 Redis、Celery、Docker、微服务。
7. 不允许覆盖 `strategies/user/`。
8. 完成后必须输出修改文件清单、接口清单、运行方式、测试结果。

## 通用测试命令

实际命令以项目初始化后的 package/scripts 为准，第一版建议固定为：

```bash
# 后端
cd backend
pytest

# 前端
cd frontend
npm run lint
npm run typecheck
```
